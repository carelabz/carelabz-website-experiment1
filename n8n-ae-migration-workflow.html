Prerequisites before building this in n8n:

1. n8n instance — Self-hosted (Docker) or n8n Cloud. Self-hosted recommended for Execute Command nodes (Lighthouse, Playwright, git).
2. Credentials to configure: Strapi API (full-access token), Vercel API token, GitHub personal access token, Google Sheets service account (for logging), Slack webhook or SMTP (for notifications), Anthropic API key (for Claude AI nodes).
3. Phase 1 must be complete first — The arc-flash-study experiment proves the stack works. This n8n workflow automates the repetitive parts for all remaining AE pages.
4. Dynamic [slug] route — Before running this, convert the static arc-flash-study/page.tsx to a dynamic [slug]/page.tsx that fetches any service page by slug from Strapi. Otherwise you'd need to generate a new static file per page.
5. Strapi schema must be finalized — The expanded ServicePage schema (with all 28+ fields from Phase 1) needs to be set before batch-running pages through this pipeline.
